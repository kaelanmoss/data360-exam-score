# Memory Updates (automatic)

Update project memory as you work. The user should not need to remind you.

## When to write

| Trigger | File to update |
|---------|----------------|
| Session starts / focus shifts | `activeContext.md` |
| Feature shipped, bug fixed, deploy done | `progress.md` |
| Chose approach A over B | `decisions.md` |
| New code pattern established | `systemPatterns.md` |
| Hosting or stack changed | `techContext.md` |
| Roadmap or vision changed | `projectbrief.md` |

## Compaction safety

If the conversation is long, re-read `activeContext.md` before making changes.
After meaningful progress, persist state to markdown immediately — do not wait for end of session.
